# Apple Docs Explorer v0.1.0

Initial public release of Apple Docs Explorer.

## What is included

- Native macOS SwiftUI app for browsing the Xcode 27 Apple Developer Documentation asset.
- Offline SQLite-backed search and framework browsing.
- Framework-name normalization for display names such as `Accessory Notifications`.
- Grouped browse sections for framework overview, topics, articles, tutorials, samples, symbols, and other entries.
- Copy actions for content and stable asset IDs.
- CLI diagnostics, search, framework listing, and framework browse commands.
- Local stdio MCP server exposing:
  - `search_apple_docs`
  - `get_doc_entry`
  - `lookup_symbol`
  - `related_docs`
- Dockerfile for CLI/MCP use in local agent workflows.

## Docker

The Docker image is for CLI/MCP usage only. It does not run the macOS GUI.

Intended GHCR image names after publishing from this tag:

```text
ghcr.io/LarryAlexander/apple-docs-explorer:0.1.0
ghcr.io/LarryAlexander/apple-docs-explorer:latest
```

Example:

```bash
docker run --rm \
  -e APPLE_DOCS_ASSET_ROOT=/docs-asset \
  -v /System/Library/AssetsV2/com_apple_MobileAsset_AppleDeveloperDocumentation:/docs-asset:ro \
  ghcr.io/LarryAlexander/apple-docs-explorer:0.1.0 diagnose-asset
```

If Docker Desktop does not allow mounting `/System/Library`, expose the asset through an allowed file-sharing path and set `APPLE_DOCS_ASSET_ROOT` to that mount.

This release includes the Dockerfile and documented CLI/MCP container workflow. If a GHCR package is not attached to the release yet, build the image locally from the tag with `docker build -t apple-docs-explorer:0.1.0 .`.

## Known limitations

- The app supports Apple's local Xcode 27 documentation asset only.
- Future Xcode asset schema changes may require updates in `DocsStore`.
- The Docker image cannot run the macOS GUI.
- The macOS app is currently signed only for local development unless a release maintainer performs distribution signing/notarization.
